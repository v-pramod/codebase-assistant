## Findings considered and rejected (do not re-audit)

- **OpenRouter `/embeddings` endpoint unsupported** — investigated and verified
  against OpenRouter docs that the OpenAI-compatible `/embeddings` endpoint *is*
  supported. Not a bug.
- **Path traversal in `files/browser.py`** — `read_file_preview` resolves the
  path and checks `root in file_path.parents`; symlink escapes are rejected.
  Safe.
- **Missing CORS / auth** — same-origin local single-user tool; authentication
  is an explicit PRD non-goal.
- **Committed `.env` with secrets** (2nd round) — `.env` is gitignored
  (`.gitignore:8-9`: `.env`, `backend/.env`), never tracked, never in history.
  Hygiene is already correct; referenced credential types: OpenRouter API key +
  Redis URL. No action beyond keeping it gitignored.
- **`VectorRecord` frozen-dataclass dict mutation** (2nd round) —
  `vector_store.py` mutates `record.metadata["active"]` on a `frozen=True`
  dataclass. Legal in Python (frozen blocks attribute rebinding, not nested
  dict mutation) and works correctly; cosmetic smell, not worth a change.
- **Prompt-injection via undelimited user/repo content** (2nd round) — real
  hardening surface (`answering.py:build_prompt`) but single-user scope makes
  the user their own adversary; low urgency. Reconsider if the tool ever gains
  multi-user or shared-session use (a PRD non-goal today).
- **git stderr returned in refresh error messages** — minor info disclosure on
  a local tool; low value.

## Deferred (surfaced, not selected for planning)

  `app/api/routes.py:27`) reconstructs state via a clones-dir + FTS scan instead
  of the PRD's SQLite data model. Real but L-effort architectural work.
- **Regex-based parsing instead of tree-sitter** (`app/chunking/chunker.py`) —
  PRD specifies tree-sitter + LangChain fallback; neither is a dependency.
  A parsing-quality spike, not a bug. **Biggest single lever on answer quality.**

  repo with a stable baseline (51 passing + 2 known `queue.py` errors), a single
  Actions workflow running `pytest`/`mypy`/`ruff` would catch executor
  regressions. Offered but not selected this round; M effort, high leverage.
- **No `CLAUDE.md`/`AGENTS.md`** (2nd round) — repo runs on `plans/` executed by
  agents but has no root guide (commands, baseline, structure). S effort.
- **Stress / adversarial evaluation** (2nd round) — golden eval covers only 4
  happy-path synthetic repos; no large-repo or no-match cases exercise
  refusal/perf at scale (PRD items 74-75). Direction/spike, not a bug.

  registry item above) — PRD §150 specifies a SQLite data model;
